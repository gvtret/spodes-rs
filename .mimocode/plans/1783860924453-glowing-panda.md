# Plan: Replace CosemDataType with Typed Attributes

## Goal

Replace generic `CosemDataType` / `Choice` / `Vec<CosemDataType>` usage in COSEM class struct fields with strongly-typed structs from the IEC 62056-6-2 (Blue Book) type definitions. This makes the code self-documenting, type-safe, and aligned with the standard.

## Constraint

The `InterfaceClass` trait's `attributes()` returns `Vec<(u8, CosemDataType)>`, so every class still converts typed fields back to `CosemDataType` at the trait boundary. This is by design — the typed structs help with internal storage, validation, and API ergonomics.

## Already Typed (no changes needed)

Clock, IecHdlcSetup, TcpUdpSetup, IecLocalPortSetup, MbusSlavePortSetup, DisconnectControl, MacAddressSetup — 7 classes fully typed.

## Phase 1: Add Typed Structs to `src/types/attrs.rs`

Add the following structs/enums, each with `From<X> for CosemDataType` and `TryFrom<&CosemDataType> for X` impls (following the existing `ScalerUnit` / `CaptureObjectDefinition` pattern):

### 1.1 Access Rights (Blue Book, Association LN)
```
AccessRight { attribute_access: Vec<AttributeAccessItem>, method_access: Vec<MethodAccessItem> }
AttributeAccessItem { attribute_id: i8, access_mode: u8, access_selectors: Option<Vec<i8>> }
MethodAccessItem { method_id: i8, access_mode: u8 }
```
Note: `security/access_rights.rs` already has `AccessRights`, `AttributeAccessItem`, `MethodAccessItem` — reuse those or unify.

### 1.2 Object List Element (Association LN attr 2)


### 1.3 Associated Partners ID (Association LN attr 3)

### 1.4 Context Name (Association LN attr 4) — CHOICE

### Note on CHOICE types

Some fields are legitimately CHOICE (any COSEM type):
- `Data.value`, `Register.value` — these stay as `Choice`
- `ProfileGeneric.buffer` — stays as `Vec<Choice>` (buffer rows are heterogeneous)
- `AssociationLN.secret` — could be `OctetString` or raw `Choice`; keep as `Choice`
- `AssociationLN.association_status` — enum, type it as `u8` (already has `AssociationStatus` enum in attrs.rs)
- `Limiter.threshold_*` — CHOICE (same type as monitored); keep as `Choice`
